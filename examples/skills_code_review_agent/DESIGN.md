# Design

本示例用代码评审 Skill 承载规则文档和受审计脚本。CLI 读取 diff、文件列表、样例或 git 工作区，解析变更文件、hunk、候选行号和 Go 包信息；规则引擎结合正则与 AST hunk 分析，覆盖安全、并发和 context、资源关闭、错误处理、测试缺失、敏感信息和数据库生命周期。

默认执行器是容器，E2B 保留入口，本地执行必须显式开启。进入沙箱前生成受控快照，过滤版本库、软链、环境文件、私钥和疑似 secret 路径；命令先过权限白名单，拒绝或需确认只记审计。沙箱限制超时、输出、环境变量、产物、网络、cap、权限、CPU、内存和 PID。

SQLite 通过存储接口保存任务、输入、沙箱运行、权限决策、finding、产物、报告和指标。finding 按文件、行号、类别去重，低置信进入 warning 或人工复核。dry-run 和 fake model 无需 API key，可验证解析、治理、落库和报告链路。
