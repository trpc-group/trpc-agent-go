A2UI rendering rules:

- Do not use Card or Row.
- Output one beginRendering message first, then one or more dataModelUpdate messages, and one final surfaceUpdate message last.
- Every A2UI message must occupy exactly one physical line.
- Do not output markdown, explanations, or any text outside those A2UI JSON objects.
- Do not emit raw newline or carriage-return characters anywhere inside JSON strings.
- Every literalString value must stay on one line. If copy would feel too long, shorten it instead of inserting a line break.
- The first message must be beginRendering for surfaceId sbti-main and root root.
- The final surfaceUpdate must be a JSON object, never a quoted JSON string.
- The final surfaceUpdate must be valid JSON. Close every object and array before ending the line.
- Prefer template-based trees over explicit repeated components. Do not inline all 30 main-question nodes or all 15 dimension nodes.
Quiz mode rules:
- In quiz mode, use dataModelUpdate to write meta copy into /session/content/meta.
- Store quiz meta copy under /session/content/meta with title, description, and notice.
- Write the 30 main questions under /session/content/questions/<question_id>.
- Every main-question item must contain display_prompt, option_1, option_2, option_3, and selected.
- display_prompt must include the visible question number followed by the exact official prompt text on one line.
- selected must equal the exact current answer value from quiz_state.answers for that question id.
- Write drink_gate_q1 under /session/content/special/drink_gate_q1 with display_prompt, option_1, option_2, option_3, option_4, and selected.
- Write drink_gate_q2 under /session/content/special/drink_gate_q2 with display_prompt, option_1, option_2, and selected.
- Keep the official full prompt text and option labels from quiz_state.questions. Never replace them with placeholder copy.
- Do not store or render hint text.
- The quiz screen must contain exactly one root Column.

- questionList must be one List component whose children.template uses componentId mainQuestionItem and dataBinding /session/content/questions.
- mainQuestionItem must be one Column component whose children.explicitList is [mainQuestionPrompt, mainQuestionChoice].
- mainQuestionPrompt must be a Text component whose text.path is display_prompt.
- mainQuestionChoice must be a MultipleChoice component whose selections.path is selected.
- mainQuestionChoice must set maxAllowedSelections to 1.
- mainQuestionChoice options must be exactly three items with values 1, 2, and 3, and labels bound to option_1, option_2, and option_3.
- drinkGateQ1Prompt must read /session/content/special/drink_gate_q1/display_prompt.
- drinkGateQ1Choice must bind selections.path to /session/content/special/drink_gate_q1/selected and render exactly four options with values 1, 2, 3, and 4.
- drinkGateQ2Prompt must read /session/content/special/drink_gate_q2/display_prompt.
- drinkGateQ2Choice must bind selections.path to /session/content/special/drink_gate_q2/selected and render exactly two options with values 1 and 2.
- Quiz option changes must stay local and must not create a network request.
- The quiz footer renders only submit_test and restart_test.
- The submit_test button must use action name submit_test.
- submit_test action.context must include exactly two top-level keys: questions and special.
- questions must use one path reference to /session/content/questions.
- special must use one path reference to /session/content/special.
- Do not flatten submit_test answers into 32 separate context entries.
- The restart_test button must use action name restart_test.
- If quiz_state.notice is empty, still render noticeText from the bound path and do not invent fallback copy.
